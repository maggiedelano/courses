The code is divided up into three sections: setup/calibration, cutting, and wrapup. For this lab, we'll focus on the cutting part, though I'll include the full code at the end if you are curious. There will be three different programs. Check the [Lab Groups Section](#LabGroups) to see which code you'll be working with. The relevant code for this lab is the `G01` G-code. This is used to cut a part using linear motion, meaning that the tool that cuts the part moves in a straight line to the specified coordinates. The `G01` code has two components: - X, Y, Z coordinates to move to - arc radius (R) or chamfer distance (C) The arc radius or chamfer distance are used to alter the corners of the part (called a fillet or a chamfer). Applying these to the corner of the piece has a number of advantages: 1) it makes assembly of multiple parts easier, 2) it keeps a user from injuring themselves on a sharp edge and 3) it looks better than a normal edge. Our code examples will apply either a fillet or chamfer to a part. ![Figure [edge]: Example of an edge that is either untouched (left), has a chamfer (middle), or fillet (right). [Image source](https://www.mytworks.com/wp-content/uploads/2019/05/Edge.Chamfer.Fillet.grey_okajtg.jpg)](images/edge_chamfer_fillet.png) An example line of G-code might look like this: `G01 Y-1., R0.0500`. This tells the code to move to -1 inches in the Y-axis, while keeping the X and Z axes constant, and with a rounded corner of 0.05 inches. If you see a `G90` after the `G01`, that means that all the following values are absolute positions for the subsequent commands. For your code example, please determine: - What you expect the size of the part to be - The likely starting position of the tool that cuts the part - What the part might look like after the program has completed You will verify your answers as you watch the code run live in the shop. It has 10 colored LEDs called neopixels that can be set to arbitrary different colors, along with sensors and buttons. You'll learn more about these in lecture. ![Figure [playground]: The Adafruit Circuit Playground.](images/circuit_playground_3000_front_lit_ORIG.jpeg) For this lab, our goal is to learn the basic program flow of the code that runs on the Circuit Playground. We'll use the neopixels as an example to get familiar with the system. The device consists of two functions that we can write code inside of. One is called `setup()`; code inside here is run once. The other is called `loop()`; code inside here is run indefinitely. When writing code for the CircuitPlayground, use the following as a template: ``` #include <Adafruit_CircuitPlayground.h> void setup(){ // code can be placed here and will run once CircuitPlayground.begin(); // initializes circuit playground functions } void loop(){ // code can be placed here and will be run repeatedly } ``` All text after a `//` are called comments. These are for humans only and are not interpreted by the Circuit Playground. In this lab, we'll focus on predicting the output of a function called `CircuitPlayground.setPixelColor()`. This function allows us to set the value of any given neopixel on the Circuit Playground to different colors of the rainbow like in the picture above. After we get familiar with the basics, we'll write our own programs as well. The `CircuitPlayground.setPixelColor()` function requires four inputs called arguments. For example, to set the 1st neopixel to red, the function would look like this: `CircuitPlayground.setPixelColor(1,255,0,0);`. Note that the arguments are contained between parentheses `()` and there is a semicolon `;` at the end that indicates the statement is complete. The arguments are as follows: 1. the neopixel number (0 to 9) 2. the intensity of **red** to set the pixel to (0 to 255) 3. the intensity of **green** to set the pixel to (0 to 255) 4. the intensity of **blue** to set the pixel to (0 to 255) In this case, we set the first argument to 1, and then the red argument to 255. This sets the 1st neopixel to the maximum possible red. Different colors can be created with different combinations of intensity for the red, green, and blue arguments. ### Running Code For The First Time To run code for the first time, log in to one of the windows computers in the lab, open the Arduino IDE program, then follow the instructions on [this page](https://learn.adafruit.com/introducing-circuit-playground/set-up-test-arduino#blink-2921627) starting with the "Blink" section.
